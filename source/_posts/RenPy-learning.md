# 开搞
## Ren'Py是什么

> Deepseek是这样说的
> 

Ren'Py 是开源、免费且拥有全球最庞大用户群的视觉小说引擎。Steam上你看到的绝大多数独立视觉小说，都是用这个引擎做的。

### 为什么适合零基础的你？

1. **脚本语言极其简单**：它的语法非常直观，几乎就像在写带有简单标记的剧本。比如，让角色说一句话，你只需要写 e "Hello, world!"。你可以在一天内就学会写出一个完整的故事。

2. **功能完备**，无需操心：它**内置了所有视觉小说需要的功能**：对话、分支选项、角色立绘、场景、音乐音效、存档读档、回滚（Rollback）等等。它甚至支持两种经典的对话模式：ADV模式（常见的底部对话框）和NVL模式（全屏文字模式）。

3. 强大的**社区和教程**：作为最流行的引擎，你能遇到的所有问题，几乎都能在中文社区或官方文档里找到答案。

4. **跨平台发布**：一次制作，可以轻松发布到 Windows、Mac、Linux、Android 甚至网页版。

### 学习门槛：

基础使用：几乎为零。

高级定制：基于 Python 语言，当你需要更复杂的游戏机制（如数值系统、小游戏）时，可以学习一些简单的 Python 知识，但这完全不是起步的障碍。

{% quote blue %}
个人体感下来，对于有计算机背景/编程能力的同学来说，门槛就是为0。而且vscode+插件就是开发环境，开源且免费就是好啊。
{% endquote %}

## RenPy项目长什么样

一个简单的RenPy项目目录结构是这样的：
```
MyVisualNovel/                          # 📁 游戏根目录
│
├── game/                               # 📁 核心游戏文件夹 (所有游戏内容都在这里)
│   │
│   ├── script.rpy                      # 📄 主脚本文件 (你的核心故事剧本)
│   ├── options.rpy                     # 📄 游戏设置 (窗口大小、标题等)
│   ├── screens.rpy                     # 📄 界面文件 (主菜单、存档界面等，通常无需修改)
│   │
│   ├── images/                         # 📁 图片文件夹 (按类型进一步分类)
│   │   ├── bg/                         # 背景图片
│   │   ├── characters/                 # 角色立绘
│   │   └── cg/                         # CG事件图
│   │
│   ├── audio/                          # 📁 音频文件夹
│   │   ├── music/                      # 背景音乐
│   │   └── sfx/                        # 音效
│   │
│   ├── chapters/                       # 📁 【可选】按章节拆分剧本
│   │   ├── chapter_1.rpy
│   │   └── chapter_2.rpy
│   │
│   ├── routes/                         # 📁 【可选】按角色路线拆分剧本
│   │   ├── route_aiko.rpy
│   │   └── route_yuki.rpy
│   │
│   └── gui/                            # 📁 游戏GUI图片 (对话框、按钮等，由GUI工具生成)
│
└── renpy/                              # 📁 Ren'Py引擎核心文件 (通常不需要改动)
```

### 每个核心部分的作用和用法

#### 游戏根目录 (MyVisualNovel/)

这个文件夹是在你用启动器创建新项目时生成的。它像一个容器，包含了你的游戏本身（game文件夹）和项目相关的其他文件（如`project.json`配置文件、`README`等）。一般情况下，你的工作重心完全在`game`文件夹内。

#### 核心文件夹 (game/)

这是整个项目的心脏，所有游戏内容都存放在这里。
   
作用：Ren'Py启动时会扫描这个文件夹，加载里面的所有脚本（.rpy文件）和资源（图片、音乐等）。

关键点：你可以在这个文件夹内自由创建子文件夹来分类管理素材，引擎会自动识别。

#### 关键脚本文件 (.rpy)
   
这是你写“代码”的地方，但实际上它更像是带了一点简单指令的剧本。

`script.rpy`：项目默认生成的主脚本文件。你可以把所有剧本都写在这里，这是最简单的做法。

`options.rpy`：存放游戏的各种配置选项，比如游戏窗口的标题、分辨率等。

`screens.rpy`：存放游戏界面的代码，比如标题菜单、对话框、存档/读档界面等。初学者可以暂时不碰它，用默认的就好。

#### 资源文件夹 (images/, audio/)

为了不让game文件夹乱成一锅粥，强烈建议用子文件夹来管理你的图片和音频。

图片：在game文件夹下新建一个images文件夹，然后在里面可以再按bg（背景）、char（角色）等进一步细分。引用时，路径就很清晰，例如 "images/bg/school.jpg"。

音频：同理，新建audio或music文件夹来存放你的BGM和音效。

### Ren'Py的脚本示例：

```python
# 定义一个角色
define e = Character("艾琳")

# 游戏从这里开始
label start:
    e "你好，欢迎来到我的视觉小说世界！"
    
    menu:
        "你感觉如何？":
            "很有趣！":
                e "太好了，我们继续吧！"
            "有点难……":
                e "别担心，一切都会很简单的。"
    
    e "我们的故事正式开始了……"
    return  # 游戏结束
```
